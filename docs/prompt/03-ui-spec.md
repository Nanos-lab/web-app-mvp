## 遇到的挑战

1. **"视觉冲击力"从何而来** — 用户给出了抽象目标但无具体方向。分析后确定：冲击力不应来自 UI 框架（过度装饰会抢内容），而应来自**内容本身的呈现质量**。因此 UI 方向定为"极致克制"——大面积留白、极简配色，让目的地的图片成为页面上唯一的视觉焦点。

2. **配色方案的平衡** — 旅行内容本身色彩丰富，UI 配色如果同样鲜艳会造成视觉疲劳。最终选择深灰底色 + 纯白卡片的"暗纸模式"，类似高端摄影画册的翻阅体验，让每张目的地图片如画廊展品般呈现。

3. **卡片比例的选择** — 16:9 和黄金比例各有利弊。16:9 与主流屏幕一致、图片素材丰富；黄金比例更具"高级感"但素材适配差。最终选择 16:9 因 MVP 阶段素材来源以 Unsplash 为主（默认 3:2 或 16:9），裁剪成本最低。预留黄金比例作为未来"精选"卡片的差异化方案。

4. **组件粒度与 API 的对齐** — 每个组件的 render prop 必须能直接从 API 响应的某个 JSON 路径取值。在文档中为每个组件绘制了 `API → 组件属性` 映射表，确保前后端契约一致。

5. **用户中断** — 该 Prompt 的 UI 规范文档实际未完成写入（仅 Writing 确认但被用户中断）。本记录总结了已完成的部分，UI 规范剩余内容待后续补充。

### 已设计的核心内容（待写入 ui_spec.md）

**视觉风格方向**：
- 配色：深色基底 `#0F0F0F`（背景）/ `#1A1A1A`（卡片）/ `#FFFFFF`（文字）
- 字体：标题用衬线体（Noto Serif SC），正文用无衬线（Inter/Noto Sans SC）
- 留白策略：卡片间距 24px，内容内边距 20px，模块间距 48-64px

**核心组件列表**：
1. **TravelCard** — 目的地卡片（列表页单元）
2. **TravelCardSkeleton** — 卡片加载骨架屏
3. **FilterBar** — 多维度筛选栏
4. **Layout** — 页面整体布局（Header + 栅格 + Footer）
5. **DestinationHero** — 详情页顶部大图区
6. **InfoSection** — 详情页信息区块
7. **ImageGallery** — 图片画廊

## 关键设计决策

- 极简主义：UI 退后，让内容（图片）说话
- "暗纸模式"：深色背景 + 白色卡片，模拟画册翻阅体验
- 16:9 作为 MVP 默认比例（素材兼容性优先）
- 悬停动效采用微缩放 + 暗色蒙层 + 文字上浮，三重反馈制造"触碰感"
